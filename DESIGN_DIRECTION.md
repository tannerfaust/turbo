# TurboTasker Design Direction

TurboTasker should feel clean and fast in the same way Linear does.

## Core Principles

- Clean, dense, calm UI. No visual clutter, no oversized cards, no decorative filler.
- Fast to scan. Information hierarchy should be obvious in under a second.
- Fast to use. Common actions should feel immediate and low-friction.
- Native desktop feel. Sharp spacing, restrained motion, strong alignment, minimal chrome.
- Quiet confidence. The interface should look serious, operational, and precise.

## Visual Rules

- Prefer neutral surfaces with one restrained accent color.
- Avoid playful gradients, soft pastel fog, and “concept app” styling.
- Use compact typography and tighter spacing.
- Reduce helper text aggressively. Only keep copy that improves decisions.
- Use color mainly for state and emphasis, not decoration.

## Interaction Rules

- Prioritize speed over novelty.
- Keep task actions close to the task.
- Make lists and tables feel crisp, not bloated.
- Default to fewer clicks and less visual noise.
- Every page should feel immediately understandable.

## Benchmark

If a screen feels slower, louder, or less precise than Linear, it is off-direction.
